# Ticket 8: Create financial goal tracking

**Description**  
The "Create financial goal tracking" feature allows users to set and monitor specific financial goals, such as saving for a vacation or paying off debt, within the Personal Finance App. This feature is integral to the app's functionality, as it empowers users to take control of their financial planning and encourages them to save and manage their finances effectively. By tracking their progress towards these goals over time, users can gain insights into their financial habits and make informed decisions that support their journey towards financial stability.

**Functionality**  
- Users can create new financial goals by specifying the goal type (e.g., saving, debt repayment) and the target amount.
- Users can set a deadline for achieving each financial goal.
- Users can input their current progress towards each goal, updating it as they save or pay off debt.
- Users can view a visual representation of their progress towards each goal, such as progress bars or percentage completed.
- Users can receive notifications or reminders related to their financial goals, such as upcoming deadlines or milestones.
- Users can edit or delete existing financial goals as their priorities change.
- Users can categorize their financial goals for better organization and tracking.
- Users can generate reports that summarize their financial goals and progress over time.

**Requirements**  
- The feature must integrate with the existing user interface of the Personal Finance App, ensuring consistency with the overall design.
- The app must allow users to input details for each financial goal, including goal type, target amount, and deadline.
- The app must store user-defined financial goals in the backend database for retrieval and tracking.
- The feature must provide visual representations of progress towards each goal, updating in real-time as users input their progress.
- The app must implement a notification system to alert users of important milestones or deadlines related to their financial goals.
- The feature must allow users to edit or delete goals, with changes reflected in the database immediately.
- The implementation must follow the design specifications outlined in Design 1-2.
- The feature must ensure that all user data is securely stored and complies with data protection regulations.
